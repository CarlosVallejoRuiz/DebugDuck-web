La historia

De dónde viene este pato

Todo empezó cuando vi a mi pareja usando GPT-4 para preguntas del día a día. No era culpa suya — nadie te explica que existe una alternativa gratuita, local y privada para el 80% de las consultas. Los tokens premium son finitos, deberían reservarse para lo que realmente los necesita.

La chispa real vino de un curso de soldadura. Sí, soldadura.

Le expliqué a mis compañeros el método del rubber duck debugging — hablarle a un objeto inanimado para ordenar tus ideas. Lo adoptaron para todo: sus proyectos, sus decisiones, su vida.

Eso confirmó algo que ya sospechaba: explicar tus problemas en voz alta funciona para todo el mundo, no solo para programadores.

DebugDuck nació de esas dos ideas.

Requisitos

¿Qué necesitas para usarlo?

Solo tres cosas. Nada raro.

✓
Mac con Apple Silicon  (M1 · M2 · M3 · M4)
✓
Windows 10 / 11  (x64)

⚠️ Si LM Studio muestra error al cargar el modelo, activa CPU only en la configuración o descarga un modelo más pequeño como phi-3-mini

✓
LM Studio o Ollama con servidor activo
✓
Micrófono funcional

Cómo configurar LM Studio en 4 pasos

1

⬇️ Descarga LM Studio

Ve a lmstudio.ai y descarga la versión para tu sistema. Es gratis y no necesitas cuenta.

2

🔍 Busca y descarga un modelo

Haz clic en el icono de lupa, escribe mistral y descarga el primer resultado. Si tienes poco RAM, busca phi-3-mini en su lugar.

3

⇄ Inicia el servidor local

Haz clic en el icono <-> Local Server, selecciona el modelo descargado y pulsa Start Server. Espera a ver Running on port 1234 ✅

4

🦆 Abre DebugDuck

DebugDuck detecta el servidor automáticamente. Sin configuración adicional — solo habla con el pato.

💡 ¿Poco RAM? Busca phi-3-mini en lugar de mistral — funciona igual de bien en equipos con 8 GB o menos.
